# Dev Note

**Automatic development documentation for Claude Code sessions.**

Dev Note installs two Claude Code hooks that silently write structured logs after every session — what was built, decisions made, current status, next steps. No commands to run. No notes to take. Just work, and find the docs waiting for you.

Designed to feed an Obsidian wiki (or any markdown-based second brain), but the output is plain markdown files — use them however you want.

---

## What it produces

```
~/claude-dev-logs/
├── my-app/
│   ├── PROJECT.md          ← living overview: stack, status, structure
│   ├── checkpoints.md      ← rolling mid-session snapshots
│   └── session-2026-04-18.md
└── another-project/
    └── ...
```

**`PROJECT.md`** — Claude maintains this in every project root. Current status, tech stack, how to run it, open questions. The single file that answers "what is this thing?" after a long break.

**`checkpoints.md`** — appended automatically when Claude Code compacts the context window (approaching token limits). Captures what happened before the history is compressed away. Zero latency — no extra LLM call.

**`session-YYYY-MM-DD.md`** — written at the end of each clean session. Claude reads the full transcript and produces a structured summary: what changed, decisions made, where things stand, what's next.

---

## How it works

Two [Claude Code hooks](https://docs.anthropic.com/en/docs/claude-code/hooks):

| Hook | Fires when | What it does |
|---|---|---|
| `PostCompact` | Context window fills mid-session | Writes `compact_summary` (already generated by Claude Code) to `checkpoints.md`. No extra LLM call. |
| `Stop` | Session ends cleanly | Calls `claude -p` with the transcript and a prompt from `document-session.md` to generate a structured session log. |

A global `~/.claude/CLAUDE.md` tells Claude to create and maintain `PROJECT.md` in every project it works in.

## Install

```bash
git clone https://github.com/yourname/dev-note.git
cd dev-note
bash install.sh
```

The installer:
- Copies hook scripts to `~/.claude/hooks/` and makes them executable
- Copies the session prompt to `~/.claude/skills/document-session.md`
- Installs global instructions to `~/.claude/CLAUDE.md` (prompts before overwriting)
- Merges hook config into `~/.claude/settings.json` (preserves your existing settings)

## Syncing to Obsidian

Edit `sync-to-vault.sh` with your server hostname and vault path, then run it from your local machine:

```bash
# All projects
bash sync-to-vault.sh

# One project
bash sync-to-vault.sh my-app
```

The logs land in `raw/dev-logs/` inside your vault. Add `ingest: true` to any file you want your wiki workflow to pick up.

For a complete Obsidian wiki setup that pairs with Dev Note, see the [LLM Wiki pattern](https://karpathy.github.io/2025/01/31/llmwiki/).

---

## Customizing the session summary

Edit `~/.claude/skills/document-session.md` to change the output format — add fields, change the structure, adjust the tone. The hook just passes this file as the prompt to `claude -p`, so it's fully in your control.

## File structure

```
dev-note/
├── install.sh                        ← one-time setup
├── sync-to-vault.sh                  ← rsync helper for Obsidian users
└── .claude/
    ├── CLAUDE.md                     ← global Claude instructions (PROJECT.md maintenance)
    ├── settings.json                 ← hook registration
    ├── hooks/
    │   ├── post-compact.sh           ← PostCompact handler
    │   └── auto-document.sh          ← Stop handler
    └── skills/
        └── document-session.md       ← session summary prompt
```

## Notes

- The `Stop` hook adds a brief delay at session end (one `claude -p` call). It fails silently and never blocks you from exiting.
- `PostCompact` is a newer Claude Code feature — test it on a throw-away session first.
- Logs are written per working directory name. If two projects share a directory name, their logs will merge. Use distinct project folder names.
- `PROJECT.md` is created by Claude in the project root, not in `~/claude-dev-logs/`. Copy or symlink it there if you want it synced.
